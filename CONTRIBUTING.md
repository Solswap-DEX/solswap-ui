# Contributing to SolSwap

First off, thank you for considering contributing to SolSwap! It's people like you that make the open source community such a great place to learn, inspire, and create.

## Developing Locally

1. Fork the repo and clone it locally
2. Create your feature branch (`git checkout -b feature/AmazingFeature`)
3. Install dependencies (`npm install`)
4. Copy `.env.example` to `.env.local` and add any necessary API keys (like BirdEye)
5. Run the development server (`npm run dev`)

## Pull Request Process

1. Ensure any install or build dependencies are removed before the end of the layer when doing a build.
2. Update the README.md with details of changes to the interface, this includes new environment variables, exposed ports, useful file locations and container parameters.
3. Your code will be reviewed by maintainers, and if it aligns with the project's goals and meets quality standards, it will be merged!
